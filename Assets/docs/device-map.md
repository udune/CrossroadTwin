# device-map.md — PLC 디바이스 맵

> **CPU**: Mitsubishi R02CPU
> **개발 환경**: GX Works3
> **최종 확정일**: 2025

---

## 1. 입력 (X)

| 디바이스 | 기능 | Unity 용도 |
|----------|------|------------|
| XA0 | 10시 방향 차량 센서 | 읽기 → 센서 상태 표시 |
| XA1 | 12시 방향 차량 센서 | 읽기 → 센서 상태 표시 |
| XA2 | 5시 방향 차량 센서 | 읽기 → 센서 상태 표시 |

---

## 2. 출력 (Y)

### 보행자 신호등

| 디바이스 | 기능 | 위치 |
|----------|------|------|
| Y0C0 | 10시 횡단보도 적색 | 10시 방향 |
| Y0C1 | 10시 횡단보도 녹색 | 10시 방향 |
| Y0C2 | 12시 횡단보도 적색 | 12시 방향 |
| Y0C3 | 12시 횡단보도 녹색 | 12시 방향 |
| Y0C4 | 5시 횡단보도 적색 | 5시 방향 |
| Y0C5 | 5시 횡단보도 녹색 | 5시 방향 |

### 차량 신호등

| 디바이스 | 기능 | 위치 |
|----------|------|------|
| Y0C6 | 10시 차량 적색 | 10시 방향 |
| Y0C7 | 10시 차량 황색 | 10시 방향 |
| Y0C8 | 10시 차량 좌회전 | 10시 방향 |
| Y0C9 | 10시 차량 녹색 | 10시 방향 |
| Y0CA | 12시 차량 적색 | 12시 방향 |
| Y0CB | 12시 차량 황색 | 12시 방향 |
| Y0CC | 12시 차량 좌회전 | 12시 방향 |
| Y0CD | 5시 차량 적색 | 5시 방향 |
| Y0CE | 5시 차량 황색 | 5시 방향 |
| Y0CF | 5시 차량 녹색 | 5시 방향 |

### 스텝모터 (차양막)

| 디바이스 | 기능 |
|----------|------|
| Y0D0 | 스텝모터 A상 |
| Y0D1 | 스텝모터 B상 |
| Y0D2 | 스텝모터 C상 |
| Y0D3 | 스텝모터 D상 |

> Unity에서 Y0D0~Y0D3는 차양막 상태 시각화에만 사용 (M62로 대체 가능)

## 4. 데이터 레지스터 (D)

| 디바이스 | 용도 | 범위 | Unity |

| T411 | K50 | 차양막 접힘 동작 5초 |

## 6. Unity SLMP 폴링 배치 구성

```
100ms 주기 배치 읽기:

① Y 디바이스 (16비트 배치):
   Y0C0~Y0CF → 1회 읽기

② M 디바이스 (개별 읽기):
   M200, M201, M203, M62

③ X 디바이스 (개별 읽기):
   XA0, XA1, XA2

④ D 디바이스 (워드 읽기):
   D0, D400

Unity → PLC 쓰기 (버튼 이벤트 발생 시):
   M50~M54: 버튼 ON 펄스
   D400:    온도값 쓰기
```
